# 超次元バトル - ルール仕様 / ゲームエンジン仕様

## 1. 基本ルール
- 全5ラウンドで構成する
- 各プレイヤーはゲーム開始時に役職を1つ選ぶ
- 各プレイヤーは基礎攻撃 / 基礎魔法を持つ
- 各ラウンド開始時、一時攻撃 = 基礎攻撃、一時魔法 = 基礎魔法
- 各ラウンド開始時、手札が7枚になるまで引く
- 各ラウンド1回だけ、任意枚数捨てて7枚になるまで引き直せる
- 各ラウンド、新規に配置して解決対象にするカードは最大5枚
- 場に保持しておけるカード枚数は無制限
- 配置後、カードを左から順に解決する
- すべてのカード解決後、最終攻撃を行う
- ラウンド終了時、場から2枚を選んで捨てる
- 5ラウンド終了後、累計ダメージ最大のプレイヤーが勝利

## 2. ステータス
### 基礎ステータス
- 基礎攻撃
- 基礎魔法

### 一時ステータス
- 一時攻撃
- 一時魔法

### ルール
- ダメージ計算に用いるのは最終的な一時ステータス
- ラウンド中に基礎攻撃が増えた場合、同量だけ一時攻撃も増える
- ラウンド中に基礎魔法が増えた場合、同量だけ一時魔法も増える

## 3. カード種別
- attack
- spell
- ability

## 4. 効果種別
- activate
- placed
- consume
- enchant

## 5. 用語定義
### 数値（ノーマル数値）
カード本体に書かれた変動数値のうち、確率数値を除いたもの。
- `+10` / `×2` / `×1.5` のような値を含む
- `風まとい` に書かれた `+10` や `×2` は数値（ノーマル数値）
- ここで言うのは「カード本体に書かれている変動数値」の分類であり、その効果が何を変動させるかとは別である

### 確率数値
カード本体に書かれた変動数値のうち、確率に関するもの。

### カード数値
カード本体に書かれた変動数値全体。  
`数値（ノーマル数値） + 確率数値` を指す。

### エンチャント数値
付与されているエンチャントに書かれた変動数値。
- `きらめく雪景色` 自体に書かれた `×2` はエンチャント数値

### ステータス数値
`数値（ノーマル数値） + エンチャント数値` を合わせた概念。

### 数値全体
カード数値とエンチャント数値を合わせた概念。

### 連続
発動順において、同じ属性が続けて解決されること。追加発動も連続に含む。

### 連結
場において、隣接またはまとまりを持つ状態。発動順とは別概念。

### 破壊
対象カードを場から除去する。

### 無効
対象カードはそのラウンド中発動しない。場には残る。

### 作成
新しいカードまたはトークンカードを場に生成する。

### 復活
破壊されたカードが指定タイミングで場へ戻る。

### 追加発動
対象カードが追加でもう一度発動する。

### 複数回発動
対象カードが1回の処理中に複数回発動する。

## 5.1 数値変動ルール
- カードの数値を増減させる効果は、原則としてカード数値にのみ影響する
- カード数値の増減は、エンチャント数値には影響しない
- エンチャント数値を増減させる場合は、別個の効果として明示する
- 「どこに書かれている変動数値か」と「その効果が何を変動させるか」は別概念として扱う
- 例:
  - `きらめく雪景色` に書かれた `×2` はエンチャント数値
  - その `きらめく雪景色` が変動させる対象はステータス数値
  - `風まとい` に書かれた `+10` や `×2` は数値（ノーマル数値）
  - `風まとい` が変動させる対象もステータス数値

## 5.2 変動数値タグ
- カード定義 JSON の `textValueBindings` には、処理側の意味づけ用に補助タグを持たせてよい
- `writtenValueKind`
  - その値がどこに書かれている変動数値か
  - `normal | probability | enchant`
- `affectsValueKind`
  - その効果が何を変動させる対象か
  - `normal | probability | enchant`
- これらは UI 表示用ではなく、仕様整理・処理分岐用の内部情報として扱う

## 6. 複数回発動 / 追加発動の特殊ルール
### 自分自身を再発動する場合
対象カード自身が自分を追加発動 / 複数回発動する場合、  
「複数回発動させる効果」そのものを除いた内容だけ再解決する。

### 他カードを再発動する場合
あるカードが別のカードを追加発動 / 複数回発動させる場合、  
対象カードのすべての要素を再解決する。  
その中に複数回発動が含まれるなら、それも含める。

## 7. 最終攻撃
- 基本的に max(一時攻撃, 一時魔法) をそのままダメージとする
- ただし役職効果や設置効果により例外がありうる

## 8. 計算順
- 加算 / 乗算 / 破壊 / 無効 / エンチャント付与は、すべて場の左から順に逐次解決する
- 一括計算しない
- 小数処理は初期実装では `floor` で切り捨て推奨（要件未確定なら設定可能にする）

## 9. エンジン設計方針
ゲームエンジンは UI から独立した純粋 TypeScript モジュールとする。

### 必要責務
- 状態遷移
- カード解決
- 効果解釈
- 発動順管理
- 連続 / 連結判定
- ログ生成
- リプレイイベント生成
- 無限ループ防止

## 10. 無限ループ防止ルール
初期実装では以下を推奨。
- 1ラウンド中の総イベント数上限を設定する（例: 5000）
- 1カードから派生する自己再発動上限を設定する
- 上限到達時は `LOOP_GUARD_TRIGGERED` ログを出し、それ以上の派生を打ち切る

## 11. 役職サンプル
### バランス
- 攻撃カードと魔法カードを使用できない
- 最終攻撃時、自分のステータスを平均化し、掛け算した結果をダメージとする

### シンプル
- 毎ラウンド7枚まで配置可能になる

### フィナーレ
- 最後のラウンドで、カードが発動するたび一時ステータス×2
## 12. 設置効果の持続ルール
- `placed` は「そのカードが場に存在している間だけ有効」が原則。
- `placed` のソースカードが、破壊・無効化後の除去・ラウンド終了時の選択廃棄などで場からいなくなったら、そのカード由来の持続効果は消える。
- 他カードに及ぶ `placed` 効果は、対象カードの実データを破壊的に書き換え続けない。
- 代わりに `derived` の一時補正や persistent aura の再計算で表現する。
- 具体例:
- `変電施設`
- 属性変更と倍率補正は、場にいる間だけ有効。消えたら戻る。
- `凍結の呪い`
- 両隣への `凍傷` 付与は、場にいる間だけ維持。消えたら剥がれる。
- 例外:
- 予約済みの次ラウンド効果、ラウンドバフ、明示的に永続状態へ書き込む仕様は別管理で残ってよい。
