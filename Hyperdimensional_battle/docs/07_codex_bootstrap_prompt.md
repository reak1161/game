# Codex向けブートストラッププロンプト

あなたは、Webブラウザ向けカードゲーム **「超次元バトル」** の実装を担当する開発エージェントです。
以下の方針を厳守してください。

## 開発目標
- React + TypeScript + Vite でフロントエンドを実装する
- ゲームエンジンは UI から独立した純粋 TypeScript パッケージとして実装する
- Cloudflare Workers + Durable Objects + WebSocket を用いて、ルーム単位の共有観戦状態を管理する
- 一画面に収まるUIを作る
- 発動時の演出として、パーティクル・効果音・テンポ加速を入れられる構造にする

## 最重要要件
1. カードや役職はデータ定義で増やせるようにする
2. カード解決は左から逐次処理する
3. 場に保持できるカード枚数は無制限
4. 「連続」は発動順、「連結」は場のつながりとして別概念にする
5. 追加発動も連続に含む
6. 自己再発動時は「複数回発動させる効果」そのものを除いて再解決する
7. 他カード再発動時は対象カードの全要素を再解決する
8. 全員が同じ再生状態を見られる共有観戦を実装する
9. ログ / ReplayEvent を重視し、演出とゲームロジックを分離する

## 実装順
- まず monorepo の雛形を作る
- 次に shared 型定義を作る
- 次に engine の最小実装を作る
- 次にローカルUIを作る
- 次に Worker / DO / WebSocket を実装する
- 最後に PixiJS / Howler.js を追加する

## 期待する成果物
- 実行可能なコード
- ディレクトリ構成
- 型定義
- サンプルカード定義
- ユニットテスト
- 共有観戦の最小実装

## 注意
- いきなり全カードを実装しない
- まずはサンプル役職3種、カード10〜15枚程度で成立させる
- ロジックを UI に埋め込まない
- Durable Object は「共有観戦の進行管理」に集中させる
- 再生中にルール再計算をしない。提出済みイベント列を再生する方式を優先する
