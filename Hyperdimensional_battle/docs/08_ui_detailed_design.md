# 超次元バトル - UI詳細設計

## 1. 目的
本書は、超次元バトルのゲームプレイ画面を一画面に収めるための詳細UI設計を定義する。

## 2. 画面全体方針
- PCブラウザ横画面前提
- 主役は自分の場と発動中カード
- 詳細情報はポップアップまたはサイドパネルで補う

## 3. プレイ画面レイアウト
## 3.1 スクロール仕様（明示）
一画面に収める方針を維持するため、各領域のスクロール挙動を明示する。

### 場エリア
- **横スクロール必須**
- カードは **折り返し表示しない**
- 左から右への発動順が見た目でも維持されることを優先する
- 場のカード保持枚数は無制限のため、表示上はスクロールで対応する
- 発動中カードは拡大表示または中央寄せ表示を許可する
- 画面外にカードがある場合、左右端に続きがあることを示すUIを出す

### 手札エリア
- **基本は1列横並び**
- 通常想定枚数では可能な限り全カードを表示する
- 画面幅不足時、または手札枚数増加時は **横スクロール対応**
- カードは **折り返し表示しない**
- ドラッグ操作とスクロール操作が競合しないよう配慮する

### ログエリア
- **縦スクロール必須**
- 最新ログを自動追従する
- ユーザーが手動で過去ログを見ている間は自動追従を一時停止する
- 新規ログ到着時は「新着あり」表示を出せるようにする
- 長文ログは折り返して表示してよい

### 右パネル詳細エリア
- 必要に応じて **内部縦スクロール可**
- カード詳細、付与エンチャント、関連ログなど、情報量が多い場合でもレイアウトを崩さない

## 3.2 スクロール操作方針
- マウスホイール / トラックパッドで自然に操作できること
- 場エリアと手札エリアは、Shift+ホイールまたは横スクロールジェスチャ対応を検討する
- スクロールバーは細めにしつつ、存在は分かるようにする
- モバイル対応時はスワイプによる横スクロールを採用する

### 上部ヘッダー
- ルームID
- ラウンド数
- 現在フェーズ
- 現在再生対象プレイヤー

### 中央メインエリア
- 自分の場

### 下部手札エリア
- 手札表示

### 右側サイドパネル
- 他プレイヤー状況
- ログ
- 詳細カード情報
