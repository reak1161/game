# ハイロール 要件定義書（入口）

このリポジトリの仕様の一次情報は `docs/requirements_design.md` です。
このファイルは「どこを見ればいいか」を迷わないための入口として、最低限の共通ルールだけを置きます。

## 共通ルール（重要）
- ドキュメント/ソースコードは **UTF-8** で統一する（Shift-JIS 等は混在させない）。
- 小数が出る計算（割合・倍率など）は、カード/ロールに明記がない限り **切り捨て（floor）**。
  - 例: `1.5` → `1`
