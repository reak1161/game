# セキュリティ対策メモ

## 実施済み

- フロントエンドでは、プレイヤー名・ログ・カード説明などの可変文字列を `innerHTML` で描画しない。
- プレイヤー名はクライアントとサーバーの両方で制御文字を除去し、24文字までに制限する。
- API / WebSocket は Origin を確認し、基本的に `https://three-turn.reak1161.com` とローカル開発環境だけを許可する。
- CORS の `Access-Control-Allow-Origin: *` は使わない。
- WebSocket メッセージはサイズ上限と短時間の送信回数上限を設ける。
- ルームコードは8文字にして、4文字コードより総当たりされにくくする。
- 公開される `playerId` だけでは再参加できないように、本人だけが持つ `playerToken` を追加する。
- Cloudflare Pages 用の `_headers` で CSP、`nosniff`、Referrer Policy、Permissions Policy を設定する。

## 残る注意点

- 本格的なアカウント認証はない。部屋コードと参加者のブラウザ内トークンを前提にした軽量な保護。
- Cloudflare Pages の `_headers` は Pages にデプロイしたときに有効になる。Worker だけを再デプロイしても静的ページのヘッダは更新されない。
- ルームコードを知っている人は新規プレイヤーとして参加できる。招待制にしたい場合は、部屋パスワードやホスト承認を追加する。
- CSP は現在の構成に合わせて絞っている。外部CDNや外部画像を追加する場合は、必要な接続先だけを個別に追加する。
